Say Hello to Java

Java was released in 1998 by Sun Microsystems and later sold to Oracle which own it til this day. It is purely Object Oriented, where everything you write has to be in a class. Java was the answer to Bell Lab's C++ and today it is used for enterprise applications, databases and other areas in tech.

At the time of it's release it was terribly slow, but today it is relativly fast in general. But it makes up for the weakness in speed by the JVM, which also is the reason for Java's drop in speed. The JVM makes it so that you don't need to recompile your code when shipping it to a new machine (that also has a JVM). That's where Java's famous catch phrase- Write once, run anywhere -comes from.

Strength of Java

  • Object-oriented
  • You don't need to re-invent the wheel thanks to the enormus Java API
  • Really secure
javac [filename].java // how to compile source code

java [filename] // how to run the compiled code

java -d . [filename].java // how to compile class that's needed for another program

Classes and Objects

When a program is Object Oriented, it is divided into diffrent parts. These parts are called Objects and together they run the program or solve the given problem. Classes can be seen as an Object's blueprint/ datatype, and every Object can be seen as an instance of a class. This basically means that the Class of an Object gives it the functionality needed to run the program / solve the given set of problems.

A thing to keep in mind is that all programs in Java are essentially classes, and that the agreed upon rule when making a class is that it should atleast have 1 method, if it isn't an inner class. But only one class in a program gets to have the main method which is the starting point of execution.

class Class {
     public static void main(String[] args) { //main method
         Scanner in = new Scanner(System.in); // instantiating an object "in" of class "Scanner"
    }
}

Methods and Variables

A variable can be seen as a box that has a type which tells us what it can contain. A method can be seen as a box that does something. It can produce a side effect or a return value using other variables and values. The most known method in Java is main, as said in the section above.

A method is made of a method head and it's body. A method head is made of the name of the method, the return-type, and parantheses, which can contain parameters. The method body is just the code that is executed when the method is invoked.

Built-in data types of Java

  • [integers] int, short, long, byte
  • [real numbers] float, double
  • [characters] char
  • [true or false] boolean
  • [sequence of characters, text] String
int num; // declaration of variable
num = 12; // assignment of variable
int otherNum = 13; // initiation of variable

int methodNum(int num1, int num2) { // definition of method
     result = num1 + num2;
     return result;
}

int res = methodNum(num, otherNum); // calls methodNum and stores return value in new variable "res"

Primitives vs. References

A primitive data type is one that is already present in the language, an example of this could be int. A reference data type is just another way of refering to an object. Primitives and reference data types works in different ways. A primitve variable is immutable while objects are mutatable. An examples of that is the fact that primitives are only copied for functions while reference types are passed.

References also refer to a type of variable who's sole purpose is to point to another object's fields. These are created by instatiating one object but only initiating the other one, which makes it the reference. When the instantiated object changes, the reference will also change, and vice versa.

StringBuilder ob = new StringBuilder("Hello"); // instantiation of object
StringBuilder ref = ob; // initiation of reference

ref.append(", World"); // mutates object through reference
System.out.println(ob); // new value: "Hello, World"

Object Oriented Principles

Inheritance: It is when a class inherits fields from other classes. The class that inherits is called a sub class and the class that is inherited from is called a super class. A sub class can only inherit from one class, but a super class can have an infinite amount of sub classes. If you'd want to inherit a class's features to another new class, you need to use the extends keyword in the class head. If a class doesn't have the extends keyword, it still inherits from the class of all classes, Object.

Encapsulation: Also called information hiding, and refers to the visability of methods, classes, and variables. Visability refers to what classes can use the methods and fields of another class. Encapsulation is implemented with access modifiers, which are the keywords private, protected and public. Not having an access modifier makes it default, which means that it is visible throughout the whole program. It is not recommended.

Polymorphism: It is when you choose to change an inherited method to do something else. It is basically method overloading, but between classes. This is also called method overriding.

Abstraction: It is when you hide unnecessary details from users. This can be great because the user doesn't need to get how things work, and the programmer can fix stuff or remake the whole thing without the user even noticing a change, because they can't see the implementation. Java makes t even easier with something called interface, that can only contains method heads with instructions on what the methods do.
